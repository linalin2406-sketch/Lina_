# 🌍 Как выложить портал в интернет (пошагово)

Инструкция рассчитана на то, что вы **не программируете**. Командная строка не
нужна — всё делается мышкой в браузере. Займёт 15–20 минут. Всё бесплатно.

Схема простая:
**GitHub** (бесплатное хранилище файлов) → **Render.com** (бесплатный хостinг,
который берёт файлы из GitHub и превращает их в работающий сайт).

---

## Часть 1. Заводим GitHub и загружаем файлы

### 1.1. Создать аккаунт
1. Откройте **https://github.com/signup**
2. Введите почту, придумайте пароль и имя пользователя.
3. Подтвердите почту по письму.

### 1.2. Создать репозиторий (папку для файлов)
1. Нажмите **«+»** в правом верхнем углу → **New repository**.
2. **Repository name**: напишите, например, `book-portal`.
3. Оставьте вариант **Public** (публичный — нужен для бесплатного хостинга).
4. Ничего больше не отмечайте, нажмите **Create repository**.

### 1.3. Загрузить файлы портала
1. На странице нового репозитория нажмите ссылку
   **«uploading an existing file»** (или кнопка **Add file → Upload files**).
2. Откройте на компьютере папку `/Users/lina/book-portal`.
3. **Перетащите в окно браузера эти файлы и папку `static` целиком:**
   - `app.py`
   - `database.py`
   - `feeds.py`
   - `fetcher.py`
   - `render.yaml`
   - `requirements.txt`
   - `README.md`
   - папку `static` (внутри: `index.html`, `style.css`, `app.js`)

   > ⚠️ **Не загружайте** файл `books.db` — база создастся в облаке сама.
   > Файл `.gitignore` можно загрузить, если он виден (необязательно).

4. Внизу нажмите зелёную кнопку **Commit changes**.

Готово — файлы в GitHub. Убедитесь, что в списке виден `app.py` и папка `static`.

---

## Часть 2. Публикуем на Render.com

### 2.1. Создать аккаунт
1. Откройте **https://render.com**
2. Нажмите **Get Started** → войдите кнопкой **GitHub**
   (это свяжет Render с вашими файлами — отдельный пароль не нужен).
3. Разрешите Render доступ к вашим репозиториям, когда он попросит.

### 2.2. Создать сервис из готовой конфигурации
В проекте уже лежит файл `render.yaml` — Render настроит всё сам.

1. В панели Render нажмите **New +** → **Blueprint**.
2. Выберите репозиторий **book-portal** из списка → **Connect**.
3. Render прочитает `render.yaml` и покажет сервис `book-portal`.
   Нажмите **Apply** (или **Create**).
4. Начнётся сборка — подождите 2–4 минуты (идёт установка библиотек и запуск).
   Когда статус станет **Live** — сайт готов.

### 2.3. Открыть сайт
Вверху страницы сервиса будет адрес вида:

**https://book-portal-xxxx.onrender.com**

Это и есть ваша публичная ссылка — открывается с любого телефона и компьютера.
При первом заходе подождите несколько секунд и, если список пуст, нажмите
**«Обновить сейчас»**.

---

## 🔁 Если варианта «Blueprint» нет — ручная настройка

Иногда Render предлагает создавать **Web Service** напрямую. Тогда:

1. **New +** → **Web Service** → выберите репозиторий `book-portal`.
2. Заполните поля:
   - **Language / Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python app.py`
   - **Instance Type**: `Free`
3. Раскройте **Advanced** → **Add Environment Variable** и добавьте:
   - `HOST` = `0.0.0.0`
   - `PYTHON_VERSION` = `3.11.9`
4. Нажмите **Create Web Service** и дождитесь статуса **Live**.

---

## ❓ Важно знать

- **«Засыпание».** На бесплатном тарифе сайт после ~15 минут без посетителей
  «засыпает». Следующий заход разбудит его за ~30 секунд (страница чуть дольше
  грузится первый раз) — это нормально для бесплатного хостинга.
- **Новости в облаке.** База в облаке обнуляется при каждом обновлении сайта, но
  портал сам заново скачивает свежие новости при запуске — так что лента всегда
  актуальная.
- **Как обновить сайт после правок.** Измените файлы у себя, загрузите их снова в
  GitHub (Часть 1.3) — Render автоматически пересоберёт и обновит сайт.
- **Свой красивый адрес** (например, `myknigi.ru`) можно подключить в настройках
  Render → **Custom Domains** (домен покупается отдельно). Помогу, если понадобится.
